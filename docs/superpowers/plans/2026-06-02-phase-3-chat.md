# Phase 3 — Instant Messaging Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build real-time 1-to-1 DMs and thematic group channels with a granular per-channel notification center (mute / mentions-only / daily digest) to replace Discord.

**Architecture:** WebSocket connection managed via a Next.js Route Handler (or Ably for managed WS). Messages persisted in PostgreSQL. Redis caches user presence and unread counts. React client holds UI state via Zustand (minimal store for chat window). Push notifications via Web Push API.

**Tech Stack:** Next.js 16, WebSockets (Ably SDK or native), Redis (Upstash), Zustand, Drizzle, Zod, web-push

**Prerequisite:** Phase 0 complete. Redis instance available.

## Modules

### 1. Data model
- DB tables: `channels`, `channel_members`, `messages`, `notification_preferences`, `push_subscriptions`
- `channels` has a `type` enum: `direct | group`
- `notification_preferences` per `(user_id, channel_id)`: `all | mentions | digest | muted`

### 2. WebSocket infrastructure
- Decision point: Ably (managed, simpler) vs. native WS Route Handler (sovereign, more complex)
- Recommend: **Ably** for MVP speed; abstract behind `src/lib/realtime.ts` so it can be swapped
- Presence tracking via Ably presence or Redis sorted set

### 3. DM chat (`/chat/direct`)
- Start a DM: find or create a `direct` channel between two users
- Message list with infinite scroll (cursor-based pagination)
- Typing indicator via presence

### 4. Channel chat (`/chat/channels`)
- Predefined channels seeded (Departmental, Regional, Workgroups)
- Admin can create/archive channels
- Member list per channel

### 5. Notification center
- Per-channel preference UI (dropdown: Tout / Mentions / Digest / Muet)
- In-app unread badge (Redis counter)
- Email digest: cron job or Clever Cloud scheduled task
